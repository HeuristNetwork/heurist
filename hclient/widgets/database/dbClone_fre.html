Cette fonction copie simplement la base de données actuelle vers une nouvelle sans aucune modification. La nouvelle base de données est identique à l'ancienne à tous égards, y compris les utilisateurs, l'accès et les pièces jointes (attention à ne pas faire de nombreuses copies de bases de données contenant de nombreux fichiers volumineux, car tous les fichiers téléchargés sont copiés). La ​​base de données cible n'est pas enregistrée auprès de l'index central Heurist même si la base de données source est enregistrée.

Si la base de données est volumineuse (par exemple, de nombreux enregistrements ou de nombreux fichiers téléchargés), le clonage prendra beaucoup de temps et votre navigateur peut devenir impatient et afficher un délai d'attente. Le clonage devrait se terminer car il s'agit d'une opération côté serveur.

Veuillez lui laisser un peu de temps (15 minutes à une heure après l'expiration du délai) pour terminer l'opération, puis vérifiez la base de données clonée. Si elle affiche le même nombre d'enregistrements que l'original, l'opération a réussi

Étant donné que cette base de données contient de nouveaux éléments structurels, vous devez l’enregistrer avant de pouvoir la cloner.

Cette base de données contient de nouvelles définitions : qui sont locales à la base de données.
Avant de pouvoir être clonées, il faut leur attribuer un identifiant global unique appelé code conceptuel.
Pour cela, il faut enregistrer la base de données. Veuillez utiliser Conception > Configuration > Enregistrer avant le clonage.
