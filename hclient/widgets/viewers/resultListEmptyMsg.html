Creating and saving filters

Filters (aka searches) are used to find information in the database and to create subsets to which different actions can be applied - editing, listing, tagging, mapping, export etc. If you are not using filters, and in particular Saved filters and Facet searches, you are not getting the best out of Heurist.

Use the search/filter field at the top of the page to find records / define a filter. The filter setup icon helps build simple filters. More complex filters can be built by following instructions linked from filter help below the Search/Filter box.

After running a filter, click the Save Filter button (which only appears after a filter has been run) to save it in the tree in the navigation panel on the left.

Simple to very complex filters, including Facet queries and Rule-based expansion of query results, can be built by right-clicking in the tree in the navigation panel on the left and selecting New, New Faceted or New Ruleset.

Defining a startup filter

A startup filter can populate this panel automatically with the set of records to which you often need immediate access. For example, tags:Favourites (for records tagged as favourites), sortby:-m (for recently modified records) or a more complex query retrieving, for instance, recent meeting notes in date order or records relating to current projects.

To set a startup filter, first create a suitable filter using the filter icon at the end of the search/filter box above, then copy the filter text from the search/filter box to the Startup filter field in Settings > Preferences (menu at top right).

You can also load Heurist directly with a filter by adding a parameter &q=xxxx to the end of the URL which you use to load Heurist (where xxxx is a valid Heurist search/filter string).
