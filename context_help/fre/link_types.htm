Structure de la base de données : types de liens

Pointeur d'enregistrement

Une simple connexion à un autre enregistrement, normalement limitée à des types d'enregistrement cible spécifiques. À utiliser lorsque le champ représente une connexion directe et est permanent, par exemple : parent, auteur, composant, lieu, période.

Les champs Pointeur d'enregistrement permettent à un enregistrement de référencer d'autres enregistrements de type spécifié qui contiennent des ensembles d'informations connexes, souvent des composants de l'enregistrement parent, par exemple : le capitaine d'un navire, le père d'une personne, une période de service militaire, un épisode éducatif, un lieu de naissance. L'enregistrement référencé peut être une entité indépendante (par exemple, un lieu, un éditeur, une personne, une œuvre) ou un groupe d'informations connexes telles que les attributs d'une personne ou d'un objet, un ensemble d'attributs qui s'appliquent à une période de temps spécifiée, ou les attributs d'une partie d'un objet ou d'un type d'objet particulier. Le type de relation est cependant implicite dans le nom du champ - père, mère, service, éducation, lieu ou composant impliquent tous une relation fixe avec l'entité parente - mais ne sont pas enregistrés par ailleurs.

Marqueur de relation

Une connexion plus complexe qui permet de spécifier le type de relation et la période de validité. À utiliser lorsqu'il existe de nombreux types de connexion possibles et/ou que les connexions ont une durée de temps, par exemple les rôles dans un événement, les relations sociales, la propriété, le mariage, l'adresse.

Les champs Marqueur de relation créent une connexion entre deux entités avec un type de relation explicite (sélectionné dans une liste déroulante), ainsi qu'une plage de dates et d'autres informations contextuelles. Ils sont particulièrement utiles lorsqu'il existe une longue liste de relations potentielles, comme les rôles des acteurs dans un événement, les relations interpersonnelles, les relations stratigraphiques, etc. Les champs de marqueur de relation ressemblent à un champ composite dans le formulaire de saisie de données mais, au lieu de créer un attribut attaché à l'enregistrement, ils créent un enregistrement de relation distinct qui peut contenir des informations supplémentaires importantes sur la relation.
