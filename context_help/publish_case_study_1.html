Heurist publishing

Case study 1: Publishing a list of research links on a web site

Suppose you want to publish a live (up-to-date) list of research project hyperlinks in your web site.

  • Tag everything you want included in your web site (including Research Projects) with a tag such as "ShowInWebSite"
  • Click "Advanced Search" on the main Heurist screen:
    • set Tags to ShowInWebSite
    • set Resource type to Research Projects
    • set Sort by to Resource title (or another field)
  • Click Search and check that the correct research projects appear in the correct order
  • Click Publish and choose the output format you want to use
  • Copy the HTML code supplied into your web page
  • et voila! your page will include live research results, like this (in this case the list is not hyperlinked):
HTML code used here:
<iframe width="95%" height="60%" frameborder="0" src="http://heuristscholar.org/heurist/published.php?pub_id=526&style=html" style="border:1px solid lightgrey;"></iframe>

The advantage of this method is the ease of maintaining the list with new entries. All you have to do when you come across a new research project or other resource and want to include it in your web site, is to tag it with the ShowInWebSite tag.

For research groups, the Heurist administrator can add special workgroup tags so that all the members of a department, a research group or an academic community - and ONLY those members - can add the tag required to publish resources to the web site.
