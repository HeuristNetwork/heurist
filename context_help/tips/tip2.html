Designing your database


Each new database is set up with pre-configured set of widely-used and useful record types (entities), such as Person, Organisation, Place, Event, Web page, Digital media item and Note, as well as standard bibliographic and mapping record types, along with all the fields and vocabularies (term/category lists) they require. At the very least this is a huge time-saver and also encourages use of shared standards; in some cases this could be all you need.


Every element can be customised through the modification and addition of fields, or deleted if not required. New record types, fields and terms can be created from scratch or borrowed from existing databases, including special template databases curated by the Heurist team.


  • Borrow (Templates)      Manage | Structure | Templates
     
         Browse a list of databases registered with the Heurist clearinghouse to find useful record types (entities), fields and vocabularies (terms).
         The first few databases in the list are a (growing) set of curated structures which are likely to be of use in specific domains. more...
     
  • Adapt (Organise)      Manage | Structure | Organise
     
         This will allow you to add, delete and modify the record types (entities) in the database, organise them into related
         groups, add and modify the fields (attributes) you record, and define the terms used to populate dropdowns. more...
     
  • Connect (Visualise)      Manage | Structure | Visualise
     
         Provides a visual representation of the links between different record types (entities) and allows the creation of
         new links between entities. Fields in a record type can also be added, edited and deleted from this view. more...
     
  • Adding data
     
         There is no need to wait until your database structure is complete. You can start adding data immediately
         and return repeatedly to add to and modify the structure without losing existing data.
