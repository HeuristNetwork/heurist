Adding data


Data can be added manually through keyboard data entry, or imported from various sources including spreadsheets (saved as CSV files), map files (KML), Zotero bibliographies, multimedia files (images, sound, video), emails (from an email server).


  • Keyboard data entry      Add Data | Keyboard | Add Record
     
         Heurist automatically builds a data entry form from the structure definitions of each record type. The form
         includes the fields (pointers, relationships) which link together records. Access to the data can be defined when
         a record is created, or changed at a later date (individually or in bulk). more...
     
  • Spreadsheets      Add Data | Import | Delimited text (CSV, TSV)
     
         Existing spreadsheets and database tables can be imported by exporting them as delimited text. This function will
         match incoming data with existing records, add or update records, and 'normalise' data where several different
         entity types exist in a single table eg. columns defining play, author, theatre and director, creating separate
         records for each. Note that it is easier, for new data, to collect separate tables for each entity type. more...
     
  • Bibliographies      Add Data | Import | Zotero synchronisation
     
         We recommend maintaining bibliographies in Zotero, a widely supported Open Source tool developed with major NEH
         funding, and synchronising this to your Heurist database. Records in each database can be linked unequivocally
         to bibliographic references, while the bibliography can be shared across multiple databases. more...
     
  • Other import methods      Add Data | Import | Multimedia, Emails, Hyperlinks, KML, FAIMS
     
         Heurist provides bulk methods for creating records which index and store file-based data (notably images, but any
         type of file can be indexed), emails from a mail server, hyperlinks from web pages and GIS data expressed as KML.
         Data can also be collected in the field/lab/archive using the FAIMS (FedArch.org) Android app, which can be
         configured directly from a Heurist database and used on multiple tablets simultaneously. more...
     
  • Using the data
     
         Heurist provides a sophisticated set of retrieval tools to allow filtering the content of a database, creating
         and reusing saved filters, reporting on and printing content, visualising content and publishing content to the web.
