Reference Types

Why use them?

HEURIST is not just an internet bookmark manager. In fact, that is a small, although useful,  part of the system. It is also (on the way to being) a full bibliographic system. The core bibliographic functionality - bibliographic definitions and efficient record storage - were developed in February/March 2006, import and export from/to EndNote Refer format should be prototyped in March 2006 (with others to follow shortly), and we should have bibliographic output formats running in April or May.

Unlike the vast majority of bibliographic systems (EndNote, RefMan, ProCite and many other 'flat file' approaches), we:

  1. store bibliographic data in a true relational database, providing web accessibility and the possibility to link directly to bibliographic information through a stable bibliographic ID;
     
  2. store bibliographic 'container' information (journal, book volume, conference proceedings, series and publisher data)  non-redundantly. This has advantages in consistency and data quality - a change in one place, such as a correction to the title of a book, changes all entries which depend on that book;
     
  3. store individual 'bookmarks' and personal notes attached to shared bibliographic records which are stored only once for all users;
     
  4. provide facilities for community review and individual commentary on bibliographic entries through attached wiki and blog entries for every reference.

How to use them

When adding a personal note or bookmarking an Internet resource, reference type is set automatically. However it can be changed in the Bibliographic Info tab of the Edit page.

Click Add note or reference at the top left of the HEURIST screen, and select a reference type from the pulldown on the popup window. Enter a title for the work. You will be taken directly to the Bibliographic Info tab of the Edit screen, and appropriate fields will be displayed for this reference type.

The type of each resource listed is shown by an icon left of the resource title in Heurist searches. The icons used are listed in the search page legend (the icons are a first cut - we hope to get a professional designer to work on a more consistent and intuitive set).

If you change reference type from the pulldown in the Bibliographic Info tab, the set of fields dispalyed will change, and HEURIST will do its best to reallocate your data to equivalent fields in the new reference type (if they exist). It does not lose data which is not displayed until you actually save the record, so you can revert or change to another reference type and recover data which has disappeared (because there was no field to hold it in the intermediate type).

How to create them

You can't! But, Heurist is more flexible than you can imagine! We have an administrative interface which allows us to create new field types and new reference types (more generally, new resource record types) in minutes. Expect to see new types of record appear in the database over the next few months - we are considering adding events, conferences, research projects, thesis topics, centres etc. as new reference (record) types. The limit is not in the software but in how broad a domain we decide to address. For them moment we are restricting ourselves to notes, internet bookmarks and bibliographic references as the most critical areas.
